# Fix CI Workflow Formatting Failure

## Problem
The CI workflow on `adam-paterson/dotfiles` failed because `nix fmt -- --check .` found two files that are not properly formatted:
- `meta.nix`
- `hosts/linux/wsl.nix`

## Root Cause
The `nixfmt` formatter (v1.2.0) requires:
1. **Multi-line list formatting**: Lists with 2+ items should have each item on its own line
2. **No trailing whitespace**: The comment in `wsl.nix` has a trailing space

## Changes Required

### 1. `meta.nix` (lines 42, 65, 75)
Three `features` lists need to be reformatted from single-line to multi-line:

```nix
# Before:
features = [ "development" "gui" ];

# After:
features = [
  "development"
  "gui"
];
```

Affected host configs:
- `macbook` (line 42)
- `wsl` (line 65)
- `macbook-intel` (line 75)

### 2. `hosts/linux/wsl.nix` (line 15)
Remove trailing whitespace from comment:

```nix
# Before (trailing space after =):
# programs.zsh.shellAliases.code = 

# After:
# programs.zsh.shellAliases.code =
```

## Implementation
Run `nix fmt meta.nix hosts/linux/wsl.nix` to auto-fix both files, then commit.

## Verification
1. Run `nix fmt -- --check .` locally to confirm formatting passes
2. Push to trigger CI and verify the workflow succeeds
